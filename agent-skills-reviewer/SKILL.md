# Agent Skills Review

Agent Skills のレビューアとして、SKILL.md と関連ファイルの品質・一貫性・実用性を評価し、改善提案を行う。日本語運用を前提とする。

## 目的
- スキルの発動精度（description）と指示の実行性を高める
- 冗長さ・曖昧さ・矛盾を排除し、保守性を上げる
- ユーザーがすぐ使える実務的なスキルにする

## 対象範囲
- パラメータで指定された `SKILL.md`（指定がない場合はカレントの `SKILL.md`）
- 上記 `SKILL.md` のフロントマター（name/description）
- 本文の指示、ワークフロー、出力形式、検証ループ
- 参照ファイル（SKILL.md から直接リンクされているもの）
- スクリプトやツール指示（存在する場合のみ）

## レビュー手順（最小フロー）
1. パラメータで指定された `SKILL.md` を開く（未指定ならカレントの `SKILL.md`）
2. フロントマター（name/description）を確認
3. 見出し構造を把握し、必要セクションを抽出（見出し一覧を表示して確認）
4. SKILL.md から直接リンクされている参照ファイルのみ確認
5. 出力フォーマットに従ってレビュー結果を作成

## レビューフロー（チェックリスト）
最小フローの品質検証用チェックリストとして使用する。
```
Review Progress:
- [ ] メタデータ妥当性（name/description）
- [ ] トリガー条件の明確さ（いつ使うか）
- [ ] 指示の具体性・過不足（自由度の適正）
- [ ] 参照構造の健全性（1階層、見通し）
- [ ] 例・テンプレの有用性
- [ ] 検証/フィードバックループの有無
- [ ] 時間依存・環境依存の記述
- [ ] セキュリティ/安全性（入力検証など）
```

## 評価基準
### 1) メタデータ
- name は小文字・数字・ハイフンのみ、64文字以内
- description は第三者視点で「何を」「いつ」使うかが明確

### 2) 指示の品質
- 冗長な一般論を避け、実務で必要な情報に絞る
- 重要なルールは強調語で明確化（MUST/必ず）
- 実行に必要な前提・依存関係を明示
- あいまい語を排除（例: 適切に/なるべく/適宜/十分に/可能な限り）

### 3) 構造と可読性
- 重要情報が上部にある
- 参照は SKILL.md から一段階のみ
- 長文参照は目次を持つ

### 4) ワークフローと検証
- 複雑タスクは段階化されている
- 検証→修正→再検証のループがある

### 5) 出力の指示
- 期待される出力形式が明示されている
- 例・テンプレがある場合は実用的

## 出力フォーマット（レビュー結果）
必ず次の構成で返す：

1. **重大な指摘**（あれば）
2. **改善提案**（優先度順）
3. **軽微な指摘**（あれば）
4. **修正案（抜粋）**（必要な箇所のみ）

各項目は箇条書きで記載し、見出しラベルは固定する。

各指摘には可能な限り、該当ファイル名と行の目安を付ける。

各指摘には「影響」「根拠」「修正方針」を次の順で1行ずつ記載する。

## ガイドライン
- 断定できない点は「要確認」と明示
- 変更による影響やトレードオフを簡潔に述べる
- 可能なら具体的な文言の差し替え案を提示
- 余計な追記より「削る提案」を優先

## 参考（判断が必要な場合）
以下のケースは特に注意：
- description が曖昧でスキルが発動しにくい
- 説明が冗長で実用情報が埋もれている
- 手順が飛躍しており再現性が低い
- 参照が深くネストされている
- 時間依存の記述が「現在」や「最新」に依存している
