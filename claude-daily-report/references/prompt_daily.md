# 構成ルール
1. **プロジェクト名**: `ディレクトリ名（プロジェクトの役割や通称）`という形式で、人間が直感的に理解できるように記載。
2. **学びの言語化**: 単なる事実（〜を習得）だけでなく、「〜ができるようになり便利になった」「〜という発見があった」という体験ベースの表現にする。
3. **参照URLの集約**: 作業中に参照したドキュメントやリポジトリのURLを、末尾にリストアップする。
