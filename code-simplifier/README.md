コードの明瞭さ・一貫性・メンテナンス性を高めるためのスキルです。**機能は一切変えず**、読みやすく保守しやすい構造へ洗練します。

## 🎯 目的
- 既存の挙動を完全に保持したまま改善する
- プロジェクト基準に沿った整形と設計判断を適用する
- 不要な複雑さを取り除き、理解しやすいコードにする

## ✅ 重視する原則
- **機能保持**: 出力・副作用・挙動を変更しない
- **明瞭性優先**: 省行よりも読みやすさを優先
- **過剰な簡素化は回避**: 巧妙すぎるワンライナーやネスト三項演算子は避ける
- **一貫性**: 命名・構造・スタイルの統一を重視
- **スコープの限定**: 最近変更された箇所に集中（指示があれば拡張）

## 🧭 適用方針（概要）
- 不要なネストや冗長な処理を整理
- 変数名・関数名を意図が伝わるものに整理
- 関連するロジックの統合、重複の排除
- 明白なコメントは削除し、必要な「なぜ」のみ残す

## ✨ 期待される成果
- 読みやすく、変更に強いコード
- プロジェクト標準に沿った一貫性
- 将来的な拡張やデバッグのしやすさ

## ⚠️ 注意事項
- 機能変更・仕様変更は行いません
- 明示がない限り、広範囲の改修は行いません

## 📦 インストール
このスキルはリポジトリ内の `SKILL.md` と `README.md` で完結します。追加の依存関係は不要です。

## 🚀 使い方
1. 対象リポジトリに `SKILL.md` を配置します。
2. Codex に本スキルを読み込ませて使用します。
